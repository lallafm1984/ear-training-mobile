섹션 3 (수정): 채점 & 결과 화면

리듬 등가 처리 반영 + 양쪽 악보 색상 표시

채점 알고리즘 — 정규화 기반

음가 정규화 → 비교 파이프라인
ScoreNote[] → 붙임줄 합산 → NormalizedNote[] → 순서 비교
Step 1: 붙임줄 합산 (NormalizedNote 변환)
입력: [C4 ♩, C4 ♪ (tie)] → 출력: [C4, 총음가=6/16]
입력: [C4 ♩.] → 출력: [C4, 총음가=6/16]
↑ 두 결과가 동일 → 정답 처리
Step 2: NormalizedNote[] 순서 비교
정답: [{C4, 6}, {D4, 4}, {E4, 8}]
답안: [{C4, 6}, {D4, 4}, {E4, 8}]
→ 각 인덱스별 pitch + totalDuration 비교
NormalizedNote 타입:
interface NormalizedNote {
  pitch: string;     // "C4", "D#5", "rest"
  totalDuration: number; // 16분음표 단위 (♩=4, ♪=2, ...)
  sourceIndices: number[]; // 원본 ScoreNote 인덱스 (색상 매핑용)
}

등가 처리 예시

문제 답안 정규화 음가 판정
♩ + ♪ (tie) ♩. (점4분) 6/16 = 6/16 ✓ 정답
♩. (점4분) ♩ + ♪ (tie) 6/16 = 6/16 ✓ 정답
𝅗𝅥 + ♩ (tie) 𝅗𝅥. (점2분) 12/16 = 12/16 ✓ 정답
C4 ♩ D4 ♩ pitch 불일치 ✗ 오답
C4 ♩ C4 ♪ 4/16 ≠ 2/16 △ 부분 (음높이만 일치)

핵심 변경사항

  • 양쪽 악보 모두 색상 표시 — 문제 악보와 답안 악보 둘 다 정답(초록)/부분(주황)/오답(빨강) 하이라이트
  • 리듬 등가 정규화 — 붙임줄 조합을 총 음가로 합산하여 점음표와 동일하게 취급
  • sourceIndices 매핑 — 정규화 후에도 원본 음표 위치를 추적하여 정확한 색상 오버레이 가능
